Difference between HTTP1.1 vs HTTP2

Versions of the Internet Protocols, Feb 23, 2024

HTTP/1.1

HTTP (Hypertext Transfer Protocol) is a data communications protocol, which is the language of web and acts as the foundation of the World Wide Web. There was a lack of standardisation and a variety of implementations following the introduction of the original version.

When HTTP/1.1 was released in 1997, several enhancements were made. Additional cache controls and Content Negotiation, which supported various languages, content encodings, and content kinds, were among the features added to HTTP/1.1.

HTTP/2

With the goal of lowering web page load latency through the use of strategies like compression, multiplexing, and prioritisation, HTTP/2 was principally developed by Google. This standardisation initiative was supported from the start by a number of browsers, including Internet Explorer, Safari, Chrome, and Opera. Since 2015, the protocol has been adopted at a large rate, with particularly high rates among new sites, partly because of the browser's support.

HTTP/1's slow response time was well-known. The protocol was becoming less effective as websites became more resource-intensive. The need to reduce latency and increase page load times grew over time. Google investigated these issues. Years later, HTTP/2—which is based on the SPDY protocol, was created to improve the Internet by addressing the intrinsic limitations of HTTP/1.1.

HTTP1.1 vs HTTP2



Major Difference

HTTP/2 uses the binary framing layer to wrap all messages in binary format while preserving HTTP semantics, such as verbs, methods, and headers, in contrast to HTTP/1.1, which maintains all requests and responses in plain text format. The underlying layer would then convert these communications into binary, even if an application level API would still provide messages in the standard HTTP formats. This guarantees that web apps developed before HTTP/2 will be able to interact with the new protocol and work normally.

The binary framing layer is the primary feature that sets HTTP/2 apart from HTTP/1.1. On the contrary, messages are sent as plain text via HTTP/1.1. Head of line blocking prevents all requests from being processed until it has all the resources it needs.


Multiplexing

HTTP/1.1 loads resources one after the other; hence, in the event that a resource is blocked, all subsequent resources are blocked as well. In contrast, HTTP/2 ensures that no resource blocks any other resource by allowing multiple streams of data to be sent simultaneously over a single TCP connection


Server push

In general, a server will only serve the content to a client device upon request from the client. Modern webpages, on the other hand, frequently require the client to request several dozen different resources, thus this strategy is not always feasible.

This issue is resolved by HTTP/2, which enables a server to "push" content to a client in advance of the client's request. In addition, the server notifies the client via message what pushed content is expected.


Header compression

Smaller files load more rapidly than larger ones. Both HTTP/1.1 and HTTP/2 compress HTTP messages to make them smaller to speed up the performance. Nevertheless, HTTP/2 removes superfluous data from HTTP header packets using a more sophisticated compression technique called HPACK.

Every HTTP packet loses a few bytes as a result. Faster loading is achieved because those bytes pile up quickly, considering the number of HTTP packets required to load even a single webpage.
