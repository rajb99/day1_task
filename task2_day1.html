Objects and it's internal representation in Javascript

February 23, 2024

JavaScript Objects

One of the data types in JavaScript is the Object type. More complicated entities and different collections are stored there. The object initializer / literal syntax or the Object() constructor can be used to create objects.

JavaScript variables are the containers for the data values. Moreover, objects are variables but objects can have several values.

JavaScript follows a basic object-based design paradigm. A property is an association between a name (or key) and a value, while an object is a collection of properties.

A reference or pointer to the reference value is assigned to variables that have been granted one. The address in memory where the object is kept is indicated by that reference, also known as a pointer. The value is not actually stored in the variables. Objects can hence be defined as an unordered collection of linked data in the form of "key: value" pairs, of primitive or reference types, in JavaScript. In the context of an object, these keys are referred to as properties or methods, respectively, and might be variables or functions.


Working with Objects

When the value of a property is a function, the property is referred to as a method. Like with many other programming languages, objects in JavaScript can be thought of as real-world equivalents. An object is a stand-alone entity with properties and a type in JavaScript.

The following code assigns many values (Fiat, 500, white) to a variable named car:

car = {type:"Fiat", model:"500", color:"white"};

Values are written as, name:value pairs (name and value separated by a colon).


Objects and properties

An object's associated variable can be used to describe one of its properties. The qualities of an object are determined by its properties. An object's properties can be accessed using a dot notation:

objectName.propertyName

A property can be defined by assigning it a value. For example, an object named myCar is created and given the properties, named make, model, and year as follows:

var myCar = new Object();
myCar.make = 'Fiat';
myCar.model = '500';
myCar.year = 2000;

The properties of an object which are unassigned are undefined (and not null).


Creating Objects in JavaScript

a. Create Objects with Object Literal, by simply defining the property and values inside curly braces as follows:

let bike = {name: 'Sporty', maker:'Hudson', engine:'900cc'};


b. Create Objects with Constructor, with help of new keyword. Constructor is nothing but a function which allows to create multiple objects of the same categor as follows:

function Vehicle(name, maker) {
this.name = name;
this.maker = maker;
}
let car1 = new Vehicle('Swift', 'Maruti');
let car2 = new Vehicle('Creta', 'Hyundai');
console.log(car1.name);    //Output: Swift
console.log(car2.name);    //Output: Creta


c. By using JavaScript keyword 'new', a new JavaScript objects is created as follows:

var person = new Object();
person.firstName = “Harry”;
person.lastName = “Cole”;
person.age = 50;


d. By using the 'Object.create()' method, as it allows to choose the prototype 'object' for the object to be created, without the need to define a constructor function, as given below:

var Animal = {
type: 'Invertebrates',
displayType: function() {
console.log(this.type);
}
};
var animal1 = Object.create(Animal);
animal1.displayType(); // Output:Invertebrates
var insect = Object.create(Animal);
insect.type = 'Insects';
insect.displayType();
// Output:Insects
